## Reporting a vulnerability

**Do not open a public GitHub issue for security reports.**

Use GitHub's private vulnerability reporting:

1. Go to the **Security** tab of this repository
2. Click **"Report a vulnerability"**
3. Fill in the details and submit

Reports are submitted as private GitHub Security Advisories, visible only to maintainers. You will receive a response within **[X] business days**.

Alternatively, email: **[your-security-email@example.com]**

---

## What to include in a report

- Which file(s), endpoint(s), or component(s) are affected
- Description of the vulnerability and potential impact
- Steps to reproduce or proof-of-concept (if applicable)
- Suggested fix or remediation (if you have one)
